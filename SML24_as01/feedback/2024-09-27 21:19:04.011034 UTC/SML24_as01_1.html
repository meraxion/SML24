def polynomial_curve_fit(D, M):
    ''' This function computes the value of a polynomial with weights w on data points x.
    
    Parameters
    ----------
    D : array
        Input dataset D.
    M : int
        The degree of the polynomial.
    
    Returns
    -------
    float
        Fitted weight vector w that minimizes the sum-of-squares function.
    '''
    # YOUR CODE HERE
    x = D[:,0]
    t = D[:,1]

    # A polynomial of degree M has M+1 weights 
    # implementation by explicitly constructing matrices

    A = np.zeros((M+1, M+1))
    for i in range(M+1):
        for j in range(M+1):
            A[i,j] = np.sum(x**(i+j))

    T = np.zeros((M+1,))
    for i in range(M+1):
        T[i,] = np.sum(t*x**i)
        
    # Alternative implementation: (not finished/tested)
    # phi = x**i for i in range(len(x))
    # A = phi.T*phi
    # T = phi.T*t

    w = np.linalg.solve(A, T)
    return w

def polynomial(x, w):
    ''' This functions computes the value of a polynomial with weights w on data points x.
    
    Parameters
    ----------
    x : float
        Set of x-coordinates for which to evaluate the polynomial.
    w : float
        Input weight vector of size M+1 (for polynomial of degree M).
    
    Returns
    -------
    float
        Values of polynomial with weights w evaluated at x.
    '''
    # YOUR CODE HERE
    y = np.zeros_like(x)
    for i, pos in enumerate(x):
        z = np.full_like(w, pos)
        for j in range(len(z)):
            z[j] = w[j]*z[j]**j
        y[i] = np.sum(z)

    return y

def polynomial_curve_fit(D, M, lmb = 0):
    ''' This functions computes the value of a polynomial with weights w on data points x.
    
    Parameters
    ----------
    D : array
        Input dataset D.
    M : int
        The degree of the polynomial.
    lmb : float, optional
        Regularization parameter for polynomial curve fitting.
    
    Returns
    -------
    float
        Fitted weight vector w that minimizes the sum-of-squares function.
    '''
    # YOUR CODE HERE
    x = D[:,0]
    t = D[:,1]

    # A polynomial of degree M has M+1 weights
    # implementation by explicitly constructing matrices
    A = np.zeros((M+1, M+1))
    for i in range(M+1):
        for j in range(M+1):
            A[i,j] = np.sum(x**(i+j))

    T = np.zeros((M+1,))
    for i in range(M+1):
        T[i,] = np.sum(t*x**i)

    I = np.identity(M+1)

    # Alternative implementation (not finished/tested):
    # phi = x**i for i in range(len(x))
    # A = phi.T*phi
    # T = phi.T*t
    w = np.linalg.solve((lmb*I)+A, T)
    return w

Exercise 1.6¶

The polynomial curve fitting procedure can be extended to the case of multidimensional inputs. Assuming an input vector of dimension $D$, namely $\mathbf{x} = (x_1, x_2, \dots, x_D)$, we can write the regression function $y$ as: \begin{equation} y(\mathbf{x}; \mathbf{w}) = \sum_{j = 0}^M \left( \sum_{n_1 + n_2 + ... + n_D = j} w_{n_1 n_2 ... n_D} x_1^{n_1} x_2^{n_2} ... x_D^{n_D} \right) \label{eqn:polynomial_multidimensional} \tag{5} \end{equation}

In the last expression, $j$ refers to the order of the polynomial terms. The inner sum is over all the combinations of non-negative integers $n_1, n_2, \dots, n_D$, such that the constraint $n_1 + n_2 + \dots + n_D = j$ holds. The terms $n_1, n_2, \dots, n_D$ correspond to the exponent for each variable $x_1, x_2, \dots, x_D$ in their respective polynomial term.

Note that if $D = 1$, the above expression simplifies to the formula in equation \eqref{yxw}. The reason the second sum disappears is that there is only one combination of the non-negative integer $n_1$ for which the constraint $n_1 = j$ holds, which means that there is only a single term to sum over.

Fitting the polynomial curve to a multidimensional input vector works analogously to the one-dimensional case. However, the number of parameters (the size of $\mathbf{w}$) becomes much larger, even when $D = 2$. Write down the general polynomial curve equation in \eqref{eqn:polynomial_multidimensional} for $D = 2$. How many parameters are needed in the two-dimensional case? Compare this to the number of parameters in the one-dimensional case.

YOUR ANSWER HERE

The general polynomial curve equation for D = 2 is: $$\begin{matrix} \mathbf{y}(\mathbf{x;w}) &= \sum_{j=0}^M \left ( \sum_{n_1 + n_2 = j} \right) w_{n_1 n_2}x^{n_1}_1x^{n_2}_2 \end{matrix}$$

Let's work this out. For $j=0$ there is one combination such that $n_1 + n_2 = j$ works out, namely $n_1=n_2=0$. For $j=1$ there are two combinations; either $n_1 = 1$ or $n_2 = 1$. For $j=2$ there are three combinations; either $n_1 = 2$ or $n_2=2$ or both $n_1=n_2=1$. So in total there are six parameters. Generalizing from this leads us to the formula $(M+1)(M+2)/2$ which is the exact same formula for how many covariance parameters are needed in the covariance matrix of the joint probability of a multivariate Gaussian Graphical Model of M dimensions, which is surely no coincidence.

Now, for whether it's a coincidence or not, these both happen to be triangular numbers. Covariance matrices are by definition square and symmetric, so you only strictly need to define the parameters on the diagonal and the lower triangle (because of the symmetry), which is, as you might expect, a triangular number. This formula for (basically) computing pairs of integers that sum to some value is more analogous to the handshake problem. In both cases you are only really interested in unique combinations.
